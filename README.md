# NIFTY Signal Pod — Quant Singularity AI-SLM Screening

A fine-tuned small language model (TinyLlama-1.1B + LoRA) that reads a NIFTY
50 options market-state snapshot and outputs a structured JSON trading signal,
wrapped in a deterministic orchestrator that applies three suppression rules.

## Step-by-step run order

### 1. Data audit (run before anything else)

```bash
python data_audit.py
```

Reads `../slm_intern_data/finetune_instructions.jsonl`, prints a full audit
report, and writes `cleaned_finetune_instructions.jsonl`.

### 2. Fine-tune on Kaggle

Upload `cleaned_finetune_instructions.jsonl` as a Kaggle dataset, then run
`kaggle_finetune.py` as a notebook on T4 GPU.  Download the adapter to
`adapters/nifty_signal_pod/`.

### 3. Run evaluation

```bash
# Without RAG
python orchestrator.py --parquet ../slm_intern_data/market_states.parquet \
                        --output results/predictions_no_rag.jsonl

# With RAG
python orchestrator.py --parquet ../slm_intern_data/market_states.parquet \
                        --output results/predictions_with_rag.jsonl --rag

# Eval suite
python eval_suite.py --predictions results/predictions_no_rag.jsonl \
                     --market_states ../slm_intern_data/market_states.parquet \
                     --output results/eval_report_no_rag.json
```

### 4. Full RAG ablation

```bash
python rag_experiment.py
```

Outputs `results/rag_ablation_report.json`.

## Signal schema

```json
{
  "direction":    "CE" | "PE" | "NEUTRAL",
  "conviction":   0.0 – 1.0,
  "horizon":      "intraday" | "next_session",
  "signal_id":    "<uuid>",
  "generated_at": "<ISO-8601 timestamp>"
}
```

On any failure (JSON parse error, model crash): NEUTRAL, conviction 0.0.

---

## Orchestrator rules

| Order | Rule | Trigger | Action |
|-------|------|---------|--------|
| 1 | ADX gate | ADX_14 < 20 | Suppress — return NEUTRAL, don't call model |
| 2 | Parse gate | JSON parse fails | Return NEUTRAL, log raw output |
| 3 | Conviction gate | conviction < 0.40 | Downgrade direction to NEUTRAL |

Every decision is logged with `reason_code` + triggering values.

## Pre-committed eval thresholds

Set before training began:

| Metric | Pass threshold |
|--------|---------------|
| Directional accuracy (non-NEUTRAL) | ≥ 52% per window |
| Schema pass rate | ≥ 99% |
| Parse failure rate | < 1% |
| Suppression rate (ADX) | ≤ 60% |
| High-VIX accuracy (VIX ≥ 18) | ≥ 48% |
